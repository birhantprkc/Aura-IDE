### 2. Conversation persistence

- New module `aura/conversation/persistence.py` with `save_conversation(history, workspace_root, title=None) -> Path` and `load_conversation(path) -> History`.
- Storage: `<workspace_root>/.aura/conversations/<ISO-timestamp>-<slug>.json`. The slug is the first ~6 words of the first user message, lowercased, non-alphanumerics replaced with `-`, truncated to 40 chars. If no user message yet, use `untitled`.
- JSON schema: `{"version": 1, "created_at": iso, "model": str, "thinking": bool, "messages": [...full history including reasoning_content...]}`.
- Save behavior: auto-save after each assistant turn completes (after `Done` event in the manager loop, before the next round if there are tool calls — so partial state during a tool loop is saved). Hook into the bridge's event handler.
- Load behavior: New menu in toolbar — "Open Conversation..." opens a file picker rooted at `<workspace>/.aura/conversations/`. Loads the history, replaces current chat view content, sets the model picker / thinking toggle to whatever was saved.
- "New Conversation" button (already exists) clears the current history but doesn't delete the saved file.
- Most-recent-on-launch: on app start, if `<workspace>/.aura/conversations/` has files, load the most recent one. Add a "Don't restore on launch" checkbox in settings (see #5). Default: restore.

### 3. Syntax highlighting in code blocks

- Add `pygments` to `pyproject.toml` dependencies.
- In `chat_view.py`, when rendering markdown content from the assistant, run a pre-pass that finds fenced code blocks (` ```lang\n...\n``` `) and replaces each with HTML produced by Pygments using the `monokai` style (or another dark style — pick one that looks good on the existing dark theme background).
- Handle unspecified language gracefully — Pygments has `guess_lexer` but it's slow; default to `text` lexer if no lang specified or if `get_lexer_by_name` fails.
- Inline code (single backticks) doesn't need highlighting — leave it as monospace styled by the theme.
- Make sure the existing markdown-rendering still works for non-code content (lists, bold, etc.). If you're using `QTextDocument.setMarkdown`, you'll need to convert to HTML first then inject the highlighted code blocks; or switch to a different renderer. Pick whichever path is less invasive.

### 4. Status bar with cost meter

- Add a `QStatusBar` to `main_window.py`.
- Left side: workspace path (truncated with ellipsis if too long), model name, thinking state (Off/High/Max).
- Right side: session token totals (`{cache_hit} hit · {cache_miss} miss · {output} out`), session cost in USD (`$0.0123`, 4 decimal places).
- Pricing constants live in `config.py`. Verify they're already there from Phase 1 (`PRICING` dict). If not, add them:
  ```python
  PRICING = {
      "deepseek-v4-flash": {"in_miss": 0.14, "in_hit": 0.003, "out": 0.28},
      "deepseek-v4-pro":   {"in_miss": 1.74, "in_hit": 0.015, "out": 3.48},
  }
  ```
- The bridge gets `Usage` events on stream completion. Forward them to the status bar via a Qt signal. Accumulate per-session totals; reset on "New Conversation".
- Cost calc: `(in_hit_tokens * in_hit + in_miss_tokens * in_miss + output_tokens * out) / 1_000_000`. Per-model — track which model produced which usage if mixing in a session.
